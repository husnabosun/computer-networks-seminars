COMMENT: A length header is required because TCP is a stream-oriented protocol; it does not have built-in message boundaries.
The application must perform "framing" to know where one command ends and the next begins. If a message is truncated 
(shorter than the header length), the server's 'while remaining > 0' loop will block, waiting for the rest of the data. 
If the header is invalid (e.g., not an integer), the server catches a ValueError, sends an 'ERR invalid length header' response, 
and closes the connection to maintain
stability and prevent a crash.
